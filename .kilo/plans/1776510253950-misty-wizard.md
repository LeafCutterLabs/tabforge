# Plan: Implement Outline Normalization Feature for tabFORGE

## Goal
Add a normalize feature that scans messy outlines in selected text, suggests corrections to preset outline levels, and allows user confirmation before applying changes.

## Feature Understanding

### Core Functionality
- **Selection-Based**: Normalizes only highlighted text blocks, not entire document
- **Preview Mode**: Shows suggested changes with visual indicators before application
- **Individual Confirmation**: Users must explicitly confirm each line change via gutter clicks
- **Preset Integration**: Uses outline level presets from settings menu

### UI Components
- **Menu Placement**: Submenu under outline mode, next to "Expand to Level"
- **Gutter Indicators**: "?" for pending confirmation → "✓" for confirmed
- **Visual Feedback**: 1px red outlines for suggested changes, green for confirmed
- **Preview Lifecycle**: Persists until save/save-all operations

### Content Processing
- **Outline Detection**: Recognizes Roman numerals, letters, numbers, bullets (-.+.*)
- **Level Analysis**: Uses indentation to determine hierarchy depth
- **Normalization Rules**: Maps levels to preset styles (I., A., 1., a., etc.)
- **Bullet Conversion**: Converts bullets to appropriate outline format
- **Numbering Continuity**: Maintains proper sequences within levels

### User Control
- **Selective Application**: Only confirmed changes get applied
- **Undo Integration**: Works with universal undo system
- **Settings Integration**: Respects existing outline preferences

## Implementation Steps

### Phase 1: Research & Design (READ-ONLY)
1. Analyze existing outline parsing logic in app.js
2. Document current outline level presets and settings structure
3. Design normalization algorithm for pattern detection and level assignment
4. Sketch UI integration points (gutter, menu, preview system)

### Phase 2: Core Algorithm Development
1. Implement outline pattern detection (regex for various formats)
2. Create indentation-based level analysis logic
3. Build normalization mapping to preset styles
4. Develop numbering continuity and reset logic

### Phase 3: UI Implementation
1. Add normalize submenu to outline mode toolbar
2. Implement gutter indicator system with click handlers
3. Create preview overlay system (red/green outlines)
4. Add selection detection and highlighting

### Phase 4: Confirmation & Application Logic
1. Build individual line confirmation mechanism
2. Implement change application with undo support
3. Add bulk confirmation options (confirm all/dismiss all)
4. Integrate with save operations to clear preview

### Phase 5: Settings & Preferences
1. Link normalization to existing outline level settings
2. Add normalization-specific preferences (auto-confirm trivial changes, etc.)
3. Update settings UI with normalization options

### Phase 6: Testing & Polish
1. Test with various outline formats and edge cases
2. Validate selection-based processing
3. Test undo/redo integration
4. Performance testing with large documents

### Phase 7: Documentation & Finalization
1. Update README with normalization feature
2. Add inline code comments
3. Final UI/UX review

## Technical Considerations
- Process large selections in chunks for performance
- Handle mixed content (outline + non-outline) gracefully
- Preserve user intent with conservative change suggestions
- Maintain compatibility with existing outline features

## Success Criteria
- Users can select text and normalize outlines with visual preview
- All changes require explicit confirmation
- Normalization respects preset outline styles
- Feature integrates seamlessly with existing outline mode
